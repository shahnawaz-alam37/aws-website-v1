# Docker: The Complete Guide

## What is Docker?

Docker is an open-source platform that enables developers to package applications and their dependencies into isolated containers, making them portable across different systems and environments. Each container is a lightweight, standalone executable unit that includes everything needed to run the application—such as code, libraries, system tools, and settings.

Containers run consistently and uniformly, independently from the host infrastructure, making it reliable for development, testing, and deployment.

## Key Points About Docker

- **Containerization Standard:** Docker introduced the industry standard for containers, making application packaging and distribution extremely portable.
- **Isolation:** Containers offer isolated environments, ensuring applications run independently and securely without interfering with each other.
- **Lightweight:** Docker containers share the machine’s OS kernel, requiring fewer resources compared to virtual machines (VMs).
- **Reproducible:** Packages code and dependencies so an application always works the same, regardless of where it's run.[^8]



## Usage in Industry

Docker is widely used by companies and teams for:

**Microservices Architecture:** Splitting applications into small, independently deployable services.<br>
**Continuous Integration/Continuous Deployment (CI/CD):** Automating software building, testing, and deployment with Docker containers.<br>
**Cloud and Hybrid Deployments:** Enabling applications to run on any cloud or on-premises infrastructure without modification.<br>
**Testing Environments:** Quickly spinning up test environments that mirror production.<br>

**Common use cases:**

* Rapid software delivery and deployment.
* Building scalable cloud-native applications.
* Modernizing legacy apps to run in new environments.
* Running reproducible experiments in data science.


## Installation

### Windows, macOS, Linux

1. Visit the official Docker website: [Docker Downloads](https://www.docker.com/products/docker-desktop).
2. Download Docker Desktop for your OS (Windows, Mac, or Linux).
3. Run the installer and follow the on-screen instructions.
4. After installation, launch Docker Desktop and verify installation with:

```sh
docker --version
```

5. Docker Desktop includes Docker Engine, Docker CLI, and Docker Compose.

**On Linux:**

- You can install Docker Engine using your package manager (e.g., `apt`, `dnf`, `yum`). See official docs for distro-specific steps.

## Simple Docker Guide

### 1. Dockerfile

A `Dockerfile` is a text file that defines the steps to build a Docker image.

```dockerfile # Example: Create a simple Node.js app container
FROM node:18-alpine       # Base image with Node.js
WORKDIR /app              # Set working directory inside container
COPY . .                  # Copy all files from current directory
RUN npm install           # Install dependencies
CMD ["node", "index.js"]  # Default command to run
```


### 2. Building an Image

From the directory containing your `Dockerfile`:

```sh
docker build -t my-node-app .
```


### 3. Running a Container

```sh
docker run -d -p 8080:8080 my-node-app
```

This command runs your app in a detached container, mapping port 8080 of your host to 8080 in the container.

### 4. Useful Commands

- List running containers:

```sh
docker ps
```

- Stop a container:

```sh
docker stop [container_id]
```

- Remove a container:

```sh
docker rm [container_id]
```

- List downloaded images:

```sh
docker images
```


### 5. Sharing Images

- Push to Docker Hub (public registry):

```sh
docker login
docker tag my-node-app username/my-node-app
docker push username/my-node-app
```

- Pull images from Docker Hub:

```sh
docker pull node:18
```
